## Storage

| Component | Technology | Notes |
|---|---|---|
| Primary store | SQLite (built-in) | Python `sqlite3` module, no ORM |
| Full-text search | SQLite FTS5 | Built into SQLite 3.9+; all modern macOS/Linux have it |
| Schema migrations | Raw SQL (hand-written) | No Alembic — schema is simple and stable |

**Verify FTS5 is available:**
```python
import sqlite3
conn = sqlite3.connect(":memory:")
conn.execute("CREATE VIRTUAL TABLE t USING fts5(content)")  # must not raise
```

## NOT in the Stack

| Technology | Reason rejected |
|---|---|
| Neo4j / NetworkX | SQLite's adjacency list is sufficient for Phase 0 graph traversal |
| SQLAlchemy / Alembic | Adds complexity; raw sqlite3 + hand-written SQL is simpler and faster |
| FastAPI / HTTP server | MCP uses stdio transport; no HTTP needed in Phase 0 |
| LangChain / LlamaIndex | No LLM layer in Phase 0 |
| Redis | No caching needed; SQLite query latency is acceptable |
| npm / Node.js | Python-first; npm `seam` name is taken by seamapi |

## Lock File Policy

After Phase 0 first successful install, pin exact versions in `uv.lock` (generated by `uv sync`). The `uv.lock` file is committed to the repo.

```bash
# First time
uv sync

# Verify lock file created
ls uv.lock

# Update all deps
uv sync --upgrade
```
