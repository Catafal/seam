The spec, in one line

Make seam start recover from a missing local index by running the existing graph-only init pipeline once, then launch the normal read-only MCP server without silently rebuilding stale indexes or polluting stdio protocol output.

Design decisions (spec was ambiguous → here's the call I made)

Decision Use public CLI tests as the TDD surface. The PRD asks for externally observable startup behavior, so the first tests drive Typer through runner.invoke(app, ["start", ...]) and monkeypatch only blocking process/server boundaries.
Decision Validate MCP support before auto-init. A missing server extra now fails before any index write because the user should not pay an implicit indexing cost only to discover the MCP dependency is absent.
Decision Send auto-init status to stderr. MCP uses stdout for JSON-RPC, so progress and failure messages are routed through a stderr console before the server starts to keep protocol bytes clean.
Decision Treat docs wording as part of the contract. Review caught two stale overclaims around freshness. README and Concepts now distinguish seam start missing-index auto-init from stale-index warnings surfaced by schema/status/read tools.

Deviations (intentional departures from the spec/convention)

Deviation None yet. The current plan stays inside the PRD's missing-index-only scope.

Tradeoffs (alternatives considered)

Tradeoff Shared helper vs. duplicated startup code. I added a private _ensure_start_index helper inside the CLI entry module. Extracting a cross-command helper with serve would add more churn because the web path has different optional-extra and browser-server concerns, while the shared indexing pipeline already prevents graph drift.
Tradeoff Warm no-egress smoke vs. clean-checkout smoke. The no-egress workflow now removes tests/fixtures/.seam before the MCP smoke so the traced path covers auto-init itself. This costs a small fixture re-index but proves the new behavior remains network-silent.

Open questions (want your confirm/revise)

Open Environment opt-out deferred. The PRD marks an env-var opt-out as useful but secondary. I kept scope to the required --no-init flag because there is no existing auto-init env convention and the flag is what installers can emit explicitly.
Open Linux no-egress remains CI-only locally. The workflow now traces a clean-checkout MCP auto-init path, but this macOS environment cannot run the exact Linux strace job. Local coverage is the real MCP smoke plus the existing egress-audit unit tests.
