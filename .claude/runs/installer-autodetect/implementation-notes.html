The spec, in one line

Add a preview-only installer autodetect plan that lets agents see supported integrations, evidence, exact would-write paths, and explicit follow-up commands without mutating files.

Design decisions (spec was ambiguous → here's the call I made)

Decision Keep autodetect outside AgentTarget. Seam impact shows AgentTarget is a broad public contract. The first implementation uses a new planner module that consumes existing target methods instead of adding abstract methods to every target.
Decision Auto preview uses compact human output. Explicit --target ... --print-config keeps the existing full content preview. Auto mode prints statuses and recommended commands because its job is selection guidance, not dumping every generated guide body.

Deviations (intentional departures from the spec/convention)

Deviation None yet. The implementation is still aligned with the PRD: CLI-only, preview-only, no new graph/index behavior.

Tradeoffs (alternatives considered)

Tradeoff Planner module vs target polymorphism. Target polymorphism would make future target-specific detection extensible, but it would widen a public ABC used across installer tests and target implementations. A planner module keeps this slice smaller and can evolve into target hooks later.

Open questions (want your confirm/revise)

Resolved Should auto preview print full guidance bodies? Resolved for this PR as no: compact human output keeps auto preview token-light, and explicit target preview remains the full-content path.
