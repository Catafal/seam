The spec, in one line

Make Seam's existing graph artifact bootstrap trustworthy and agent-answerable: persist safe bootstrap provenance, expose artifact readiness through schema/status, add strict fetch failure semantics, and document the trust contract without adding graph edges or MCP mutation.

Design decisions (spec was ambiguous → here's the call I made)

Decision Use a small JSON provenance file under .seam/. The PRD allowed a metadata file or table. A file keeps bootstrap provenance outside the graph database, avoids migrations, and lets artifact import/fetch update trust metadata even before opening the landed DB.
Decision Make seam fetch --strict additive. Existing fetch behavior stays available for older artifact stores, while strict mode is the recommended automation path. Strict mode requires checksum + manifest identity before the remote index can replace local .seam/.

Deviations (intentional departures from the spec/convention)

Deviation No standalone doctor/status command in this slice. The PRD's goal is satisfied through existing schema transports. A future doctor command can consume the same bootstrap block, but this implementation avoids adding a new command surface before the contract proves useful.

Tradeoffs (alternatives considered)

Tradeoff Status vocabulary first, broader doctor later. A standalone seam doctor would be useful, but the PRD's highest leverage is one artifact/bootstrap block inside existing schema/status surfaces. The doctor command remains a follow-up, not part of this build.
Tradeoff Hash raw remotes instead of storing them. The manifest can contain sensitive repository URLs. The persisted provenance keeps only remote_fingerprint, which is enough to correlate identity without leaking internal URLs or credentials.

Open questions (want your confirm/revise)

Open Should permissive fetch become deprecated later? This implementation preserves permissive fetch for compatibility but labels it explicitly as permissive/unverified when checksum or manifest evidence is absent. Deprecation can be decided after teams have a migration window for artifact stores.
