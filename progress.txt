#       without flattening the normal spread). Deterministic, never raises.

# WS2b — ANN acceleration for semantic search via sqlite-vec
# (issues #267/#268/#269/#270, branch feat/ws2b-ann-search)
# Session: 2026-07-03
# ─────────────────────────────────────────────────────────────────────────────
#   WHY this phase exists:
#     WS2a mmap eliminated per-query blob-decode overhead but remained O(N)
#     exhaustive cosine-scan.  On ≥50k-symbol repos this is 200–500 ms/query.
#     WS2b wires in sqlite-vec vec0 as a forward-compatible scaffold for when
#     sqlite-vec ships a true ANN index (HNSW/IVF).
#
#   BENCHMARK FINDING (2026-07-03, sqlite-vec v0.1.9, synthetic 384-dim):
#     sqlite-vec v0.1.9 has NO approximate index — vec0 is EXACT brute-force KNN.
#     Measured results:
#       N=10k:   numpy BF 0.6ms  vs  vec0 3.9ms  (speedup 0.2×, recall@10 1.000)
#       N=50k:   numpy BF 5.9ms  vs  vec0 20.5ms  (speedup 0.3×, recall@10 1.000)
#       N=100k:  numpy BF 7.1ms  vs  vec0 41.6ms  (speedup 0.2×, recall@10 1.000)
#       N=250k:  numpy BF 19.7ms vs  vec0 109.4ms (speedup 0.2×, recall@10 1.000)
#     vec0 is ~5× SLOWER than numpy matmul at every scale.  SEAM_VEC_ANN is
#     shipped off-by-default and documented as a scaffold, NOT a perf feature.
#
#   Shipped slices:
#     S1 — seam/query/vec_extension.py (leaf): probe_vec_extension() — full
#       CREATE/DROP round-trip probe (no side effects on real DB); load_vec_extension()
#       — loads the extension onto a given connection for reuse.  Never raises; False
#       on any failure.  Controlled by SEAM_VEC_ANN=on guard in callers.
#     S2 — seam/indexer/vec_index.py (bridge, mirrors cluster_index pattern):
#       index_vec() — triple-gated public entry (0=skip, -1=error, ≥1=rows).
#       _build_vec_index() — inner impl (may raise): CREATE vec_embeddings vec0 table
#       + batch-INSERT from embeddings WHERE model=? + write staleness token to
#       vec_meta.  Called by seam init/sync --semantic after index_embeddings.
#     S3 — seam/query/semantic.py three-tier cascade: ANN (_try_vec_path) → mmap
#       (_try_mmap_path) → SQL brute-force.  Staleness guard reads vec_meta token,
#       compares to compute_index_version; mismatch → None (fall through).  Transparent
#       to all MCP tool callers.
#     S4 — benchmarks/semantic_ann_scale.py: run_scale() + ScaleResult dataclass +
#       CLI harness (--sizes, --dim, --queries, --k).  Self-skips when sqlite_vec
#       absent.  `make bench-semantic-ann` target added to Makefile.  NOT gate.
#       tests/unit/test_semantic_ann_bench_smoke.py: gate-wired smoke (tiny scale,
